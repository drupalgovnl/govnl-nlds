## Wanneer gebruik je de Table component

Gebruik de Table component om gebruikers makkelijk data te laten vergelijken in rijen en kolommen.

## Wanneer gebruik je de Table component niet

Gebruik de Table component niet om een layout op te zetten op een pagina, hiervoor is de [Grid component](?path=/docs/componenten-grid--docs).

## Richtlijnen

- Gebruik een `<caption>` element om te beschrijven wat er in de tabel staat, zodat het voor gebruikers makkelijk is om de tabel te begrijpen.
- Gebruiker table headers (`th`) om aan gebruikers te vertellen wat de inhoud is van elke rij en kolom.

## Voorbeelden

Een basis tabel heeft een caption met daarin de beschrijving van de tabel. De tabel begint met een `thead` element, waarin de table headers voor elke kolom in zijn opgenomen.
Deze headers beschrijven wat de inhoud is van de kolommen. Om duidelijk te maken dat het gaat om headers voor de kolommen, wordt op het `th` element een `scope` attribuut met als waarde `col` gezet.

<Canvas of={TableStories.DefaultTable} />

### Tabel met headers voor rijen

Een tabel kan naast headers voor de kolommen ook headers voor de rijen hebben. Deze headers hoeven niet in de `thead` worden opgenomen, maar kunnen ook aan de `tbody`
toegevoegd worden. De headers voor de rijen beschrijven wat de inhoud van de betreffende rij is. Om duidelijk te maken dat het gaat om headers voor de rijen, wordt op het `th`
element een `scope` met als waarde `row` gezet.

<Canvas of={TableStories.RowHeaderTable} />

### Tabel met een footer

Aan de tabel kan een `tfoot` element worden toegevoegd. De footer van een tabel bevat informatie over de bovenliggende kolommen. Vaak is dit een samenvatting van de kolom, of de totale som van bovenliggende kolom.

<Canvas of={TableStories.FootTable} />

### Tabel met cellen over meerdere rijen of kolommen

Het is mogelijk om cellen over meerdere rijen of kolommen te laten lopen. Hiervoor voeg je aan het `td` element een `colspan` attribuut toe voor kolommen, en/of een `rowspan` attribuut voor rijen.
De waarde van het attribuut is het aantal kolommen of rijen dat overbrugt moet worden.

### Tabel met uitlijning aan het einde van de cel

Standaard wordt de inhoud van een cel uitgelijnd aan het begin. Om de inhoud van een cel uit te lijnen aan het einde van de cel voeg je de classname `dictu-table__cell--align-end` toe aan de `td` element.

<Canvas of={TableStories.AlignEndTable} />

### Tabel met dubbele headers

Het is mogelijk om voor zowel row headers als voor column headers een dubbele header toe te voegen. De eerste header is dan de overkoepelende header voor de headers die op de rij eronder of de kolom erna komen.
Gebruik hiervoor de `rowspan` en `colspan` die eerder in deze documentatie beschreven zijn.
